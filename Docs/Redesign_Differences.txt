Redesign Differences

Goal
- Keep the redesign small and focused: improve model, view, and controller boundaries without rewriting the whole project.

Main changes
- Composite: slide content is treated as a tree with `Slide`, `SlideItem`, `TextItem`, `BitmapItem`, and `CompositeSlideItem`.
- Observer: `Presentation` notifies `SlideViewerComponent` when the current slide changes.
- Command: user actions are wrapped in command objects such as `NextSlideCommand`, `PrevSlideCommand`, `GotoSlideCommand`, and `ExitAppCommand`.

Why this matters
- Composite removes type checks when rendering or traversing slide content.
- Observer keeps the UI in sync with the model.
- Command keeps controller code simple and makes input actions easy to extend.

Where to look
- Model tree: `src/Slide.java`, `src/SlideItem.java`, `src/TextItem.java`, `src/BitmapItem.java`, `src/CompositeSlideItem.java`
- View update: `src/Presentation.java`, `src/SlideViewerComponent.java`
- Actions: `src/Command.java`, `src/NextSlideCommand.java`, `src/PrevSlideCommand.java`, `src/GotoSlideCommand.java`, `src/ExitAppCommand.java`, `src/KeyController.java`, `src/MenuController.java`

Short summary
- The redesign uses Composite for structure, Observer for updates, and Command for actions. That reduces coupling and makes the code easier to read, test, and extend.
